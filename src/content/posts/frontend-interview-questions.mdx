## 一、HTML / CSS 篇（20 题）

### 1. 盒模型有哪几种？如何切换？

- 标准盒模型（`content-box`）：`width` 只含内容区；
- IE 盒模型（`border-box`）：`width` 含 content + padding + border；
- 工程实践：全局 `*, *::before, *::after { box-sizing: border-box; }`。

### 2. CSS 选择器有哪些？优先级怎么算？

- 类型：元素、类、ID、属性、伪类、伪元素、后代/子代/相邻选择器；
- 优先级：`!important` > 行内 > `#id` > `.class/伪类/属性` > 标签/伪元素 > 通配符；
- 同权重后写的覆盖先写的。

### 3. BFC 是什么？触发条件与应用？

- 独立渲染区域，内外布局互不影响；
- 触发：`overflow: hidden/auto`、`display: flow-root`、`float`、`position: absolute/fixed` 等；
- 应用：清除浮动、阻止 margin 合并、自适应两栏。

### 4. 水平垂直居中方案（元素定宽高 / 不定宽高）

定宽高：`flex` / `grid` / `absolute + transform` / `absolute + margin: auto`；
不定宽高：`flex` / `grid` / `table-cell`（少用）。

### 5. 两栏布局（右侧自适应）与三栏布局（中间自适应）

- 两栏：`float + margin-left` / `flex` / `grid`；
- 三栏经典：圣杯、双飞翼、`flex: 1` 中间列。

### 6. Flex 与 Grid 的区别与适用场景

- Flex：一维布局（行或列），适合导航、卡片对齐；
- Grid：二维布局，适合整页栅格、复杂仪表盘。

### 7. `flex: 1` 是什么缩写？

`flex: 1 1 0%` → grow/shrink/basis；与 `flex: auto`（basis: auto）分配剩余空间行为不同。

### 8. `em` / `rem` / `vh` / `vw` 区别

- `em`：相对当前元素 font-size；
- `rem`：相对根元素 font-size；
- `vh/vw`：视口高宽 1%；移动端注意地址栏导致 100vh 问题。

### 9. 隐藏元素的几种方式及区别

`display: none`（不占位、不可访问）、`visibility: hidden`（占位）、`opacity: 0`（占位、可响应事件）、`clip/aria-hidden` 等；SEO 与可访问性不同。

### 10. 回流与重绘？如何优化？

- 回流（Reflow）：几何变化，代价大；重绘（Repaint）：外观变化；
- 优化：`transform/opacity` 动画、批量改 class、DocumentFragment、虚拟列表。

### 11. 响应式设计原理与常见方案

流式布局 + 弹性盒 + 媒体查询；移动端 `viewport` meta；`clamp()` / 容器查询（Container Queries）。

### 12. 单行 / 多行文本溢出省略

```css
/* 单行 */
.ellipsis { overflow: hidden; text-overflow: ellipsis; white-space: nowrap; }
/* 多行 */
.clamp { display: -webkit-box; -webkit-line-clamp: 3; -webkit-box-orient: vertical; overflow: hidden; }
```

### 13. CSS 画三角形原理

利用 border 透明 + 一边有色形成三角形；也可用 `clip-path`。

### 14. CSS 预处理器（Sass/Less）区别

变量、嵌套、Mixin、函数；Less 运行时编译，Sass 有 Scss 语法；PostCSS 是后处理插件生态。

### 15. CSS 性能优化手段

减少选择器层级、避免 `@import`、关键 CSS 内联、减少重排属性动画、合理使用 `will-change`。

### 16–20. 速览题

| 题 | 答题要点 |
|----|----------|
| 设备像素 / DPR / PPI | 物理像素 vs CSS 像素，Retina 屏 DPR=2/3 |
| `position` 各值 | sticky 需 top + 父级无 overflow:hidden |
| 层叠上下文 | z-index 仅在同层叠上下文比较 |
| 伪类 vs 伪元素 | `:hover` vs `::before` |
| `@media` vs `@container` | 视口响应 vs 容器响应 |

## 二、JavaScript 基础篇（25 题）

### 1. JS 数据类型？存储差别？

- 原始：string、number、boolean、null、undefined、symbol、bigint；
- 引用：object（含 Array、Function、Date 等）；
- 原始存栈（快），引用存堆、栈存指针。

### 2. `var` / `let` / `const`

函数作用域 vs 块级作用域、提升与 TDZ、重复声明、`const` 锁引用。

### 3. 闭包

函数 + 词法环境；场景：私有变量、防抖节流、柯里化；注意内存泄漏。

### 4. 原型链与 `instanceof`

`obj.__proto__` 链式查找；`instanceof` 沿链查 `Constructor.prototype`。

### 5. 继承方式

原型链、构造函数、组合、寄生组合（推荐）、ES6 `class extends`。

### 6. `this` 指向规则

new > 显式 bind/call/apply > 隐式 > 默认；箭头函数取词法 this。

### 7. 执行上下文与执行栈

创建阶段（VO/LexicalEnvironment）→ 执行阶段；调用栈 LIFO。

### 8. 事件循环（Event Loop）

同步 → 微任务清空 → 一个宏任务 → 微任务…；`async/await` 后续是微任务。

### 9. 事件模型：捕获、目标、冒泡

`addEventListener` 第三参 `true` 捕获；`stopPropagation` / `preventDefault` 区别。

### 10. 事件委托（代理）

利用冒泡把监听绑在父节点；动态列表、减少监听器数量。

### 11. `new` 操作符做了什么？

创建对象 → 链式原型 → 绑定 this 执行构造函数 → 返回对象（构造函数无 return 对象时）。

### 12. `call` / `apply` / `bind` 区别

`call(thisArg, ...args)`、`apply(thisArg, [args])`、`bind` 返回新函数；手写 bind 注意 `new` 场景。

### 13. 深拷贝 vs 浅拷贝

`Object.assign`、展开是浅拷贝；`structuredClone`、递归 + WeakMap 处理循环引用。

### 14. 防抖与节流

防抖：停止触发后执行（搜索）；节流：间隔内最多一次（scroll）。

### 15. `typeof` vs `instanceof`

`typeof null === 'object'` 历史 bug；`instanceof` 查原型链，跨 iframe 失效。

### 16. 内存泄漏常见场景

未清理的定时器/监听器、闭包引用 DOM、全局变量、 detached DOM 树。

### 17. 本地存储对比

Cookie（4KB、随请求发送）、localStorage、sessionStorage、IndexedDB。

### 18. 尾递归

函数最后一步调用自身，引擎可优化栈；JS 引擎优化有限，了解概念即可。

### 19. 函数缓存（memoize）

用 Map 存参数 → 结果；斐波那契、纯函数计算。

### 20. 数字精度丢失

0.1 + 0.2 !== 0.3；用整数运算、`decimal.js`、展示层 `toFixed`。

### 21. 判断元素是否在可视区域

`IntersectionObserver`（推荐）、`getBoundingClientRect` + scroll 计算。

### 22. 大文件上传断点续传

分片（chunk）+ 哈希校验 + 服务端合并 + 断点记录（localStorage/服务端）。

### 23. 上拉加载 / 下拉刷新

scroll 监听 + 节流；或 `IntersectionObserver` 哨兵元素；移动端 touch 事件。

### 24. 常见 Web 攻击与防御

XSS（转义、CSP）、CSRF（Token、SameSite）、点击劫持（X-Frame-Options）、SQL 注入（后端）。

### 25. AJAX 原理

`XMLHttpRequest` / `fetch`；异步请求、回调 → Promise；CORS 跨域。

## 三、ES6+ 与异步篇（12 题）

### 1. 数组新增方法

`map/filter/reduce/find/findIndex/includes/flat/flatMap/at`；可变：`push/pop/shift/unshift/splice`。

### 2. 对象扩展

`Object.assign`、展开、简写属性/方法、计算属性名、`Object.keys/values/entries`。

### 3. 函数扩展

默认参数、剩余参数、箭头函数（无 arguments、无 own this）。

### 4. Promise 状态与静态方法

`all` / `race` / `allSettled` / `any`；链式 then、错误穿透 catch。

### 5. `async/await` 本质

Generator 语法糖；并发用 `Promise.all`，不要循环里串行 await。

### 6. Generator

`function*` + `yield`；可暂停；co 库自动执行；迭代器协议。

### 7. Proxy 与 Reflect

Vue 3 响应式基础；拦截 get/set/has/deleteProperty 等；Reflect 配套调用默认行为。

### 8. Set / Map / WeakSet / WeakMap

Set 去重；Map 键可为对象；WeakMap 弱引用、不可遍历，适合私有数据。

### 9. Symbol 应用场景

唯一属性键、`Symbol.iterator`、内置 Symbol（如 `Symbol.hasInstance`）。

### 10. Module（ESM vs CJS）

静态分析、tree-shaking、循环依赖处理、动态 `import()`。

### 11. Decorator（了解）

类/方法/属性装饰；TC39 提案；TypeScript 实验支持。

### 12. 可选链与空值合并

`?.` 短路访问；`??` 仅 null/undefined 触发默认值（与 `||` 不同）。

## 四、浏览器原理与性能篇（10 题）

### 1. URL 到页面展示全过程

DNS → TCP/TLS → HTTP → 解析 HTML/CSS → 构建 DOM/CSSOM → Render Tree → Layout → Paint → Composite。

### 2. 强缓存 vs 协商缓存

`Cache-Control: max-age` vs `ETag/If-None-Match`；304 与 200 from cache。

### 3. `defer` vs `async`

`defer`：HTML 解析完再执行，保序；`async`：下载完即执行，不保序。

### 4. 跨域方案

CORS、代理、JSONP（历史）、postMessage、nginx 反向代理。

### 5. 性能优化体系

加载（分割、CDN、压缩）、渲染（SSR、虚拟列表）、交互（防抖、Worker）；Core Web Vitals：LCP、INP、CLS。

### 6–10. 速览题

| 题 | 要点 |
|----|------|
| 浏览器进程与线程 | 浏览器多进程，渲染进程主线程 + 合成线程 |
| 同源策略 | 协议+域名+端口 |
| Service Worker | 离线缓存、Push，PWA |
| 首屏优化 | 骨架屏、关键资源 preload、SSR/SSG |
| 长列表 | 虚拟滚动、分页、IntersectionObserver |

## 五、网络 / HTTP 篇（15 题）

### 1. HTTP vs HTTPS

HTTPS = HTTP + TLS；加密、完整性、身份（证书）。

### 2. HTTP 1.1 / 2 / 3

1.1 长连接；2 多路复用、HPACK；3 QUIC(UDP) 解决 TCP 队头阻塞。

### 3. TCP 三次握手 / 四次挥手

SYN → SYN+ACK → ACK；FIN 双向关闭；TIME_WAIT 作用。

### 4. UDP vs TCP

UDP 无连接、不可靠、低延迟（视频、DNS）；TCP 可靠有序。

### 5. DNS 查询过程

浏览器缓存 → OS → 路由器 → ISP → 递归/迭代查询根/顶级/权威服务器。

### 6. CDN 原理

边缘节点缓存静态资源，就近访问，回源。

### 7. 常见状态码

301/302/304、401/403/404、500/502/503/504。

### 8. GET vs POST

语义、幂等、缓存、Body、预检触发条件。

### 9. 常见请求头

`Content-Type`、`Authorization`、`Cache-Control`、`Accept`、`Cookie`、`Origin`。

### 10. WebSocket

HTTP Upgrade 握手；全双工；心跳与重连设计。

### 11–15. 速览题

| 题 | 要点 |
|----|------|
| OSI 七层 / TCP/IP | 应用层 HTTP 在最上 |
| 对称 vs 非对称加密 | HTTPS 混合加密 |
| 粘包拆包 | TCP 流式，需定界（长度头） |
| HTTP 缓存策略 | private/public/no-cache/no-store |
| 单点登录 SSO | CAS、OAuth、Token 共享域 |

## 六、Vue 2 篇（30 题）

> Vue 2 相关面试仍常见，尤其 legacy 项目与 Options API 对比题。

### 1. 说说你对 Vue 的理解？

渐进式框架：模板/响应式/组件化/生态（Router、Vuex/Pinia）；MVVM 思想，数据驱动视图。

### 2. SPA 单页应用的理解与优缺点

优点：体验流畅、前后端分离；缺点：首屏慢、SEO 弱（可 SSR 补）；hash/history 路由。

### 3. `v-show` vs `v-if`

`v-if` 条件渲染（DOM 销毁/创建）；`v-show` 切换 `display`，适合频繁切换。

### 4. 为什么 `v-if` 和 `v-for` 不建议一起用？

`v-for` 优先级更高，每个 item 都走一遍 `v-if`，性能差；应用 computed 过滤后再 v-for。

### 5. Vue 实例挂载过程发生了什么？

`new Vue` → 初始化 initLifecycle/initEvents/initRender → `beforeCreate` → initInjections/initState → `created` → `$mount` → 编译 template → 首次 render → `beforeMount` → patch DOM → `mounted`。

### 6. Vue 生命周期（常用钩子）

创建：`beforeCreate/created`；挂载：`beforeMount/mounted`；更新：`beforeUpdate/updated`；销毁：`beforeDestroy/destroyed`（Vue3 为 `beforeUnmount/unmounted`）；`activated/deactivated`（keep-alive）。

### 7. 为什么 `data` 必须是函数？

组件复用时每个实例需要独立数据对象；根实例可以是对象。

### 8. 响应式原理（Vue 2）

`Object.defineProperty` 递归劫持；数组通过重写 `push/pop/splice` 等方法；新增属性需 `Vue.set`；依赖收集 Dep + Watcher。

### 9. 给对象添加新属性界面不刷新？

Vue 2 无法检测新增属性；用 `this.$set(obj, key, val)` 或替换整个对象。

### 10. 双向绑定原理

`v-model` = `:value` + `@input`（组件上是 `:modelValue` + `@update:modelValue`）；底层仍是单向数据流 + 事件回写。

### 11. `$nextTick` 理解

DOM 更新异步批量 flush；在下次 DOM 更新后执行回调；底层 Promise > MutationObserver > setImmediate > setTimeout。

### 12. 组件通信方式

props / `$emit`；`$parent/$children`（不推荐）；`provide/inject`；Vuex/Pinia；EventBus（Vue3 移除）；`$refs`；URL/query。

### 13. `computed` vs `watch` vs `methods`

computed 缓存依赖；watch 副作用；methods 每次渲染重新执行。

### 14. `key` 的作用

diff 时标识节点身份，复用 DOM；不要用 index 作 key（增删排序会乱）。

### 15. 虚拟 DOM 与 diff 算法

同层比较；tag/key 不同则替换；双端比较 + 最长递增子序列（Vue3）；key 影响复用策略。

### 16. `keep-alive` 缓存组件

`<keep-alive include/exclude/max>`；`activated/deactivated` 钩子；缓存的是 VNode 不是 DOM。

### 17. slot 插槽

默认插槽、具名插槽、`v-slot`、作用域插槽（子传数据给父模板）。

### 18. mixin 理解与应用

选项合并；混入生命周期都会执行；缺点：命名冲突、来源不清晰；Vue3 推荐 composables。

### 19. 自定义指令

`Vue.directive('focus', { inserted(el) { el.focus() } })`；DOM 操作类场景。

### 20. 修饰符

`.stop/.prevent/.capture/.self/.once`；按键 `.enter`；表单 `.lazy/.number/.trim`。

### 21. SPA 首屏慢怎么优化？

路由懒加载、骨架屏、CDN、Gzip、SSR/预渲染、减小 bundle、图片优化。

### 22. Vue 项目封装 axios

实例 + 拦截器（Token、错误统一处理）；取消重复请求；类型封装。

### 23. Vue 权限管理（路由 + 按钮）

路由：`meta.roles` + 全局守卫；按钮：`v-permission` 自定义指令 + 权限列表。

### 24. Vue 项目跨域

开发：`devServer.proxy`；生产：Nginx 反向代理；CORS 后端配置。

### 25. 部署后刷新 404

history 模式需服务器 fallback 到 `index.html`；Nginx `try_files $uri /index.html`。

### 26. Vue 错误处理

`errorCaptured`、全局 `Vue.config.errorHandler`、axios 拦截、Sentry 上报。

### 27. 组件 vs 插件

组件 UI 单元；插件 `Vue.use()` 扩展全局功能（Router、Vuex、UI 库）。

### 28. 过滤器（Vue 3 已移除）

纯函数格式化展示；Vue 3 用 computed / 方法替代。

### 29. 大型 Vue 项目目录结构

`views/components/api/store/router/utils/assets`；按业务域拆分；公共组件抽离。

### 30. Vue 2 vs Vue 3 核心区别（过渡题）

Proxy 响应式、Composition API、Fragment/Teleport/Suspense、移除 filter/EventBus、性能与 tree-shaking。

## 七、Vue 3 篇（15 题）

> Vue 3 是当前主流，Composition API 与性能优化是重点。

### 1. Vue 3 设计目标与优化

更小更快、TS 友好、Composition API、模块化（按需引入）、Proxy 响应式。

### 2. 性能提升体现

Proxy 替代 defineProperty、静态标记 patchFlag、Block Tree、编译优化（hoistStatic）、tree-shaking。

### 3. 为什么用 Proxy 替代 defineProperty？

可监听新增/删除属性、数组索引、Map/Set；lazy getter；性能更好；API 更简洁。

### 4. Composition API vs Options API

逻辑按功能组织 vs 按选项组织；复用 composables vs mixins；更好的 TS 推导；`setup` 执行时机在 beforeCreate 之前。

### 5. `setup` 函数要点

无 `this`；返回对象/template 可用数据；异步 setup 需 `<Suspense>`；生命周期用 `onMounted` 等。

### 6. `ref` vs `reactive`

`ref` 包装基本类型，`.value` 访问；`reactive` 对象代理；解构 reactive 会丢失响应式 → `toRefs`。

### 7. `watch` vs `watchEffect`

watch 需指定源；watchEffect 自动收集依赖，立即执行；`watchPostEffect` / flush 时机。

### 8. Vue 3 Tree-shaking

按需引入 API（`import { ref } from 'vue'`）；未使用代码打包时移除。

### 9. `defineComponent` / `defineProps` / `defineEmits`

`<script setup>` 编译宏；类型推导；运行时零开销。

### 10. Pinia vs Vuex

Pinia 无 mutations、TS 友好、模块化更简单、支持 Vue 2/3；Vuex 4 兼容 Vue 3。

### 11. Teleport / Suspense / Fragment

Teleport 传送 DOM；Suspense 异步组件加载；Fragment 多根节点。

### 12. Vue 3 自定义 Hook（Composable）

抽离响应式逻辑，如 `useMouse`、`useFetch`；注意依赖注入与 SSR 水合。

### 13. 实现 Modal 组件设计思路

`Teleport` 到 body；遮罩 + 内容区；`v-model:visible`；focus trap；ESC 关闭；滚动锁定。

### 14. Vue 3 SSR（Vite SSR / Nuxt）

服务端 renderToString → 客户端 hydrate；注意状态同步、只读数据、生命周期差异。

### 15. Vue 3 + TypeScript 实践

`<script setup lang="ts">`；props/emits 类型；组件实例类型 `InstanceType<typeof Comp>`。

## 八、React 篇（30 题）

> React 面试侧重 Fiber、Hooks、状态管理与性能优化。

### 1. 说说你对 React 的理解？

UI 库，声明式、组件化、单向数据流；虚拟 DOM + diff；函数组件 + Hooks 为主流。

### 2. Real DOM vs Virtual DOM

VDOM 是 JS 对象描述 UI；批量 diff 更新；权衡可维护性与性能下限，非绝对更快。

### 3. React 生命周期（类组件）

挂载：`constructor → render → componentDidMount`；更新：`render → componentDidUpdate`；卸载：`componentWillUnmount`；错误：`componentDidCatch`。

### 4. 函数组件生命周期对应

`useEffect(() => { mount/update; return unmount }, [deps])`；空 deps 仅 mount+unmount。

### 5. `state` vs `props`

props 父传子只读；state 组件内部可变；都不可直接修改，需 setState / 新对象。

### 6. `setState` 是同步还是异步？

React 18 自动批处理；合成事件/setTimeout 内也可能批量；`flushSync` 强制同步。

### 7. `super()` 和 `super(props)` 区别

后者先初始化 props，constructor 里可访问 `this.props`。

### 8. React 事件机制

合成事件（SyntheticEvent）委托到 root；池化（React 17+ 取消）；与原生事件冒泡顺序：原生 → React → 原生。

### 9. 事件绑定方式

JSX `onClick`；类组件需 bind 或 class fields；避免 render 里 bind 产生新函数。

### 10. 组件构建方式

函数组件（推荐）、类组件；高阶组件 HOC、Render Props（旧模式）。

### 11. React 组件通信

props 回调、Context、Redux/Zustand/Jotai、组合 children、全局 EventEmitter（少用）。

### 12. `key` 的作用

diff 标识；index 作 key 在重排增删时状态错乱；稳定唯一 id。

### 13. refs 理解与应用

`useRef` 持久化可变值、访问 DOM、保存上一值；`forwardRef` 转发 ref。

### 14. 类组件 vs 函数组件

函数 + Hooks 更简洁、易测试、无 this；类组件仍有 Error Boundary（函数可用 react-error-boundary）。

### 15. 受控 vs 非受控组件

受控：value 由 state 控制；非受控：DOM 自身管理，ref 取值；表单推荐受控。

### 16. 高阶组件 HOC

`const Enhanced = withX(Wrapped)`；注意 displayName、ref 转发、不要 render 里创建 HOC。

### 17. React Hooks 解决了什么问题？

类组件逻辑难复用、生命周期割裂、this 问题；Hooks 按功能组织、自定义 Hook 复用。

### 18. Hooks 规则与原理

仅顶层调用；Fiber 链表存 hook 状态，顺序不能变；`useState/useEffect/useMemo/useCallback/useRef/useContext/useReducer`。

### 19. `useEffect` vs `useLayoutEffect`

Effect 异步 paint 后；LayoutEffect 同步 paint 前，适合测量 DOM、防闪烁。

### 20. 闭包陷阱

effect/定时器拿到旧 state；依赖写全、函数式更新、`useRef` 存最新值。

### 21. `useMemo` / `useCallback`

缓存计算结果 / 函数引用；配合 `React.memo` 避免子组件无效 render；不要过度优化。

### 22. React 引入 CSS 方式

CSS Modules、CSS-in-JS（styled-components）、Tailwind、全局 CSS；CSS Modules 作用域隔离。

### 23. React 过渡动画

`react-transition-group`、Framer Motion、CSS transition + 条件渲染。

### 24. Redux 理解与工作原理

单一 store、reducer 纯函数、dispatch action → 新 state；单向数据流。

### 25. Redux 中间件

redux-thunk（异步）、redux-saga（generator）、redux-observable；洋葱模型。

### 26. React Router 理解与模式

BrowserRouter / HashRouter；`useNavigate/useParams/useLocation`；路由守卫用 wrapper 组件。

### 27. React diff 原理

同层比较；type/key 不同替换；单节点 vs 多节点 diff；key 优化列表移动。

### 28. Fiber 架构解决了什么？

可中断渲染、优先级调度、时间切片；reconciliation 与 commit 两阶段；为 Concurrent Mode 铺路。

### 29. JSX 转真实 DOM 过程

JSX → `React.createElement` → VDOM 对象 → Fiber 构建 → diff → commit 操作 DOM。

### 30. React 性能优化手段

`React.memo`、useMemo/useCallback、懒加载 `React.lazy + Suspense`、虚拟列表、避免 Context 大范围更新、Profiler 分析。

## 十、手写题篇（12 题）

### 1. 手写 Promise.all / race / allSettled

```js
function promiseAll(list) {
  return new Promise((resolve, reject) => {
    const res = [];
    let n = 0;
    const arr = Array.from(list);
    if (!arr.length) return resolve([]);
    arr.forEach((p, i) =>
      Promise.resolve(p).then((v) => {
        res[i] = v;
        if (++n === arr.length) resolve(res);
      }, reject),
    );
  });
}
```

## 十二、复习建议

1. **按模块刷，Vue / React 分开**：别混在一个「框架篇」里记，面试常只深挖一个方向；
2. **每题准备 30 秒 + 3 分钟两版答案**：30 秒讲结论，追问再展开原理；
3. **Vue 2 项目经验 + Vue 3 新特性都要会**：很多公司问「从 2 迁到 3 你怎么做」；
4. **React 必过 Hooks + Fiber + 性能**：三板斧；
5. **手写题动手写**：Promise、并发控制、深拷贝出现率极高；
6. **结合项目**：axios 封装、权限、部署 404、跨域 — 有真实经历加分最多。
